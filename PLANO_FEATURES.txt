[x] 1. Notificação de check-in online (24h antes)
    Arquivo: TransitMark/State/NotificationScheduler.swift
    - Adicionar em scheduleAll(for:) um novo lembrete por voo:
      triggerAt = scheduledDeparture - 24h
      title = "Check-in disponível"
      body  = "<displayCode> · <origin> → <destination>"
    - Garantir que a janela seja > .now (voo ainda não partiu).

[x] 2. Duração do voo + progresso em tempo real (InFlightView)
    Arquivo: TransitMark/Views/InFlightView.swift
    - Calcular duração total: scheduledArrival - scheduledDeparture
    - Calcular progresso: (now - scheduledDeparture) / duração  (0…1)
    - Mostrar barra ProgressView linear + "Chegada em Xh Ymin"
    - Usar TimelineView(.periodic(by: 60)) já presente no ContentView
      ou receber `now: Date` via parâmetro (preferível — sem relógio
      interno na view).

[x] 3. Rota do aeroporto → hotel (InFlightView + LandingView)
    Arquivos: TransitMark/Views/InFlightView.swift
              TransitMark/Views/LandingView.swift
    - Substituir o botão "Copiar endereço" por dois botões:
        • "Copiar endereço" (mantém comportamento atual)
        • "Como chegar"  → MKMapItem.openMaps com origem no aeroporto
          de destino (flight.destinationCoordinate) e destino na Stay.
    - Usar MKMapItem(location:address:) — API iOS 26 (sem MKPlacemark).
    - Se Stay não tiver coordenada, abrir apenas o endereço como texto.

[x] 4. "Abrir no Mapas" nas linhas de Stay e Activity (TripDetailView)
    Arquivo: TransitMark/Views/TripDetailView.swift
    - Em StayRow: adicionar swipe action .leading "Navegar"
      (systemImage: "map.fill") quando stay.coordinate != nil.
    - Em ActivityRow: idem quando activity.coordinate != nil.
    - Ação: abrir MKMapItem com a coordenada do item.
    - Alternativa (menor esforço): context menu com Label("Como chegar").

[x] 6. Hora local do destino no StayingView
    Arquivo: TransitMark/Views/StayingView.swift

    - Atualizar via TimelineView no pai (ContentView já tem .periodic(30)).

[x] 7. Compartilhar itinerário
    Arquivo: TransitMark/Views/TripDetailView.swift

      atividades) em ordem cronológica.
    - Expor via ShareLink ou UIActivityViewController.
    - Adicionar ao menu "…" já existente na toolbar de TripDetailView.

[x] 8. Importação por câmera / arquivo (boarding pass)
    Arquivo novo: TransitMark/State/BoardingPassImporter.swift
    - Usar VisionKit (DataScannerViewController) ou PHPickerViewController
      para capturar imagem / PDF.
    - Passar o texto extraído para FoundationModels (on-device LLM) com
      um prompt estruturado para extrair: companhia, número do voo,
      origem, destino, horários, assento, portão, código de reserva.
    - Pré-preencher FlightEditorView com os dados extraídos.
    - Ponto de entrada: botão "Importar boarding pass" em TripDetailView.
